## 🎯 슬라이드 3 — 프로젝트 개요 `[0:45 ~ 3:05]` ⏱ **1분 40초**

왼쪽의 **문제 정의**를 먼저 보겠습니다.

현재 대부분의 카드·금융 앱은 소비 내역 조회에만 그칩니다.
"이번 달 식비 30만 원" — 숫자만 보여줄 뿐, 이게 많은 건지 적은 건지, 어느 항목을 줄여야 하는지 알려주지 않습니다.

이상 거래가 발생해도 문제는 마찬가지입니다. 탐지 자체가 늦고, 사용자는 피해가 발생한 뒤에야 알게 되는 경우가 많습니다.

세 번째로, 개인화된 예산·절약 조언이 없습니다. 누군가는 식비를 줄여야 하고, 누군가는 쇼핑을 줄여야 하지만 모든 사람에게 같은 가이드라인이 제시됩니다.

네 번째, 소비 패턴 분석과 사기 탐지가 연동된 서비스가 사실상 없습니다. 소비 이력을 보면 이상 거래를 훨씬 더 정확하게 잡아낼 수 있는데, 지금은 이 둘이 완전히 분리되어 있습니다.

오른쪽 **솔루션**입니다.

저희는 이 네 가지 문제를 AI로 해결합니다.
K-Means 군집화로 사용자 소비 유형을 절약형·균형형·소비형·투자형, 네 그룹으로 분류하고, 유형에 맞는 맞춤 조언을 제공합니다.
Isolation Forest로 이상 거래를 실시간으로 탐지하고, LSTM과 XGBoost로 다음 달 지출을 예측하거나 과소비 여부를 분류합니다.
마지막으로 규칙 엔진과 ML 모델을 결합한 하이브리드 사기 탐지 구조로, 단순 임계값 규칙보다 훨씬 정교하게 위협을 판단합니다.

하단에 보이는 네 아이콘 — AI 소비 패턴 분석, 실시간 사기 탐지, 개인화 예산 전략, 소비 유형 군집화 — 이 네 가지가 PayWise의 핵심 목표입니다.

## 🎯 슬라이드 4 — 시스템 아키텍처 `[3:05 ~ 4:35]` ⏱ **1분 30초**

전체 시스템 구조입니다.

크게 **세 개의 레이어**로 나뉩니다.

프론트엔드는 **Next.js + React + TypeScript** 조합입니다. Recharts를 활용해 소비 패턴을 게이지·레이더·바 차트로 시각화하고, 사용자가 직접 분석 결과를 확인할 수 있도록 구성했습니다.

백엔드는 **FastAPI**로 구현했습니다. 비동기 처리에 강점이 있어 ML 추론과 DB 쿼리를 동시에 처리하는 데 적합합니다. 데이터 저장은 PostgreSQL, 토큰과 캐시는 Redis, DB 스키마 변경은 Alembic 마이그레이션으로 자동 관리합니다.

ML 엔진은 K-Means, Isolation Forest, LSTM, XGBoost 네 모델을 학습 파이프라인으로 묶어 운영합니다.

핵심 설계 포인트는 **fraud-service를 별도 서비스로 분리**한 부분입니다. 포트 8010에서 사기 탐지 추론만 전담합니다. 사기 탐지 모델은 교체 주기가 잦고 스케일이 독립적으로 필요하기 때문에 메인 백엔드와 분리해서 각자 독립적으로 배포·확장할 수 있도록 설계했습니다.

이 전체 구성을 **Docker Compose** 하나로 통합 관리합니다. 향후 JWT 인증 완성과 CI/CD 구성도 이 인프라 위에 올릴 예정입니다.

## 🎯 슬라이드 5 — AI 소비 패턴 분석 `[4:35 ~ 6:05]` ⏱ **1분 30초**

핵심 AI 기능 네 가지를 설명드리겠습니다.

첫 번째, **K-Means 소비 유형 군집화**입니다.
식비, 쇼핑, 교통 등 11개 카테고리별 지출 비율을 특성으로 사용합니다. 데이터를 표준화한 뒤 K=4로 클러스터링해 사용자를 절약형, 균형형, 소비형, 투자형으로 분류합니다. 같은 소비 유형끼리는 같은 조언이 더 효과적이기 때문에 이 군집 정보를 예산 추천의 기반으로 활용합니다.

두 번째, **Isolation Forest 이상 거래 탐지**입니다.
라벨 없이 학습하는 비지도 방식으로, "평소와 다른 패턴"을 이상 거래로 탐지합니다. contamination 파라미터 스윕 실험을 통해 탐지율과 오탐율 사이의 최적 지점을 찾았고, 거래 건마다 0~1 사이의 이상 스코어를 실시간으로 산출합니다.

세 번째, **LSTM 지출 예측**입니다.
최근 3개월 소비 시계열 데이터를 학습해서 다음 달 카테고리별 지출액을 예측합니다. "다음 달 식비가 이 정도 나올 것 같으니 미리 예산을 조정하세요" 수준의 선제적 조언이 가능해집니다.

네 번째, **XGBoost 과소비 분류**입니다.
과소비 레이블 기반 지도 학습 분류기로, 단순히 과소비 여부를 판단하는 것을 넘어 feature importance를 통해 "이 달 외식비가 전체 지출에서 가장 큰 위험 요인이었다"는 식으로 구체적인 원인까지 설명합니다.

하단 파이프라인처럼, 소비 데이터 수집 → 특성 추출 → 네 모델 추론 → API 응답 순서로 순차 처리됩니다.

## 🎯 슬라이드 6 — 사기 탐지 서비스 `[6:05 ~ 7:35]` ⏱ **1분 30초**

사기 탐지 서비스, fraud-service입니다.

왼쪽 **판단 로직**을 보시면, ML 스코어를 기준으로 네 단계로 분류합니다.
0.95 이상이면 **BLOCK** — 즉시 거래 차단입니다. 0.35 이상이면 **REVIEW** — 담당자 검토 대상으로 Step-up Auth를 자동으로 트리거합니다. 0.005 이상이면 **SOFT_REVIEW** — 경미한 의심으로 모니터링하고, 그 미만이면 **PASS**입니다.

단순히 ML 스코어만 보지 않고 **규칙 엔진**을 함께 사용합니다. 금액 이상 급등, 새벽 시간대 거래, 단시간 반복 결제인 VELOCITY, 큰 금액을 여러 번 나눠 결제하는 SPLIT_TXN 패턴 등 다차원 규칙을 평가합니다. ML 판단과 규칙 판단 중 더 강한 쪽을 최종 action으로 채택하는 구조입니다.

오른쪽에 보이는 것처럼 세 가지 모델을 비교 실험했습니다.
**Isolation Forest** 비지도 모델은 Kaggle creditcard 데이터셋으로 학습했습니다. **Random Forest** 지도 모델은 오픈 데이터셋으로 학습한 분류기입니다. **Hybrid IF+RF**는 Isolation Forest의 이상 스코어를 추가 피처로 삼아 RF로 최종 분류하는 방식으로, 단일 모델 대비 정밀도를 높입니다. 세 모델의 PR-AUC와 FPR을 비교한 결과를 CSV로 정리해두었습니다.

## 🎯 슬라이드 7 — 부가 기능 `[7:35 ~ 8:20]` ⏱ **45초**

> **⚠️ 7분 35초 이후. 여기서부터 속도를 조금 높이세요.**

여섯 가지 부가 기능을 간략히 소개합니다.

**XAI**는 feature importance에 z-score를 결합해 "이번 거래가 왜 의심스러운지" 상위 3개 이유를 사용자에게 설명합니다. 블랙박스 AI가 아닌, 설명 가능한 AI를 지향합니다.

**FIDO 인증**은 REVIEW 이상 판정 시 생체 인증으로 추가 본인 확인을 요청하는 Step-up Auth 구조입니다.

**감정 기반 분석**은 소비 패턴과 감정 데이터를 연결해 충동 소비를 감지하고 경고합니다.

**예산 전략 추천**은 소비 유형에 맞춘 카테고리 한도와 절약 팁을 제공하며, 예산 80% 도달 시 알림을 발송합니다.

**금융 교육 콘텐츠**는 분석 결과를 토대로 개인 맞춤형 투자·저축 가이드를 제공합니다.

**보안 대시보드**는 이상 거래 Push 알림과 위험 지표를 실시간으로 시각화합니다.
